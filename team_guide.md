# My Pivot — チーム共有用開発・動作ガイド

このプロジェクトの現在の状態、実装された機能、および各自のローカル環境で動かすための手順をまとめました。

---

## 🌟 実装された機能

### 1. 🔐 ユーザー管理 & ログイン機能 (新規実装)
* **JWT認証方式**: 安全なトークンベースのログイン機能を導入しました。
* **パスワードハッシュ化**: `bcrypt` を使用して安全にパスワードをハッシュ化してデータベースに保存しています。
* **データ分離**: ログインしているユーザー自身が作成したピボット（迷い）データのみが、本棚やタイムラインに表示されるように制御されています。
* **新規登録 / ログイン画面**: フロントエンドに洗練されたログイン画面を追加し、未ログイン時は自動的にこの画面へリダイレクトされます。

### 2. ⏳ 五里霧中モード & ピボット・タイムライン (バグ修正・安定化)
* **リアルタイム入力**: 左側の「五里霧中モード」から、タイトル、悩み、感情タグ、確信度を入力して保存できます。
* **バグ修正 (シリアライズとCSSの競合)**:
  * タイムライン取得時のタグのシリアライズ漏れによる画面クラッシュを完全に修正。
  * グローバルCSSと各ページのCSSの変数競合を解消し、表示崩れを直しました。

### 3. 📚 地球の書庫 v3 (新規統合)
* **「地球の書庫 v3」をタブで完全再現**: 美しい本棚のUIや、本を開くペラペラとした3Dアニメーション、SVGによる見込みと結果の「判断キャリブレーション曲線」、類似する悩みを呼び出す「召喚（Recall）」機能をReactコンポーネントとして完全に移植しました。
* **データ分離**: こちらはデモ体験用に、シードデータを保持しつつブラウザの `localStorage` で独立して動作します。

## 💻 各自のローカルでの起動手順

事前準備として、リポジトリをクローンまたは最新状態（`git pull`）にした後、以下の手順を実行してください。

### Step 1: バックエンドのセットアップと起動

1. **必要なライブラリのインストール**:
   ```bash
   cd backend
   pip install -r requirements.txt
   ```

2. **デモ用データベースの初期化 & データ投入**:
   データベース構成がユーザー対応になったため、以下のスクリプトを走らせて初期データとデモユーザーを登録します。
   ```bash
   PYTHONPATH=. python seed.py
   ```
   * ※ これにより、デモ用のアカウントとタイムラインのテスト用データが `my_pivot.db` に生成されます。

3. **サーバーの起動**:
   ```bash
   uvicorn app.main:app --reload
   ```
   * 起動後、ブラウザで `http://localhost:8000/docs` を開くとAPIドキュメント（Swagger UI）を確認できます。

💡 **エラーが出た場合（Port 8000 が既に使われている）**:
`Address already in use` というエラーが出た場合は、バックエンドの古いプロセスが裏で残っています。以下のコマンドでプロセスを終了させてから再度起動してください。
* **Mac**: `kill -9 $(lsof -t -i:8000)`
* **Windows**: `taskkill /f /im python.exe`

---

### Step 2: フロントエンドのセットアップと起動

1. **必要なパッケージのインストール**:
   別のターミナルを開き、フロントエンドのディレクトリへ移動して実行します。
   ```bash
   cd frontend
   npm install
   ```

2. **開発サーバーの起動**:
   ```bash
   npm run dev
   ```
   * 起動すると `http://localhost:5173`（競合時は `http://localhost:5174` 等）でアプリが立ち上がります。
